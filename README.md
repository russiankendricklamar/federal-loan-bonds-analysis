# Bond Portfolio Analytics

Инструмент для анализа портфеля облигаций с фокусом на российские государственные облигации и данные Московской биржи. Скрипт собирает рыночные данные, считает ключевые риск-метрики, моделирует стресс-сценарии и формирует готовый Excel-отчёт с таблицами и графиками.

## Что умеет

Проект покрывает полный базовый цикл аналитики облигационного портфеля:

- расчёт позиционной аналитики по бумагам;
- расчёт агрегированных метрик портфеля;
- стресс-тестирование через параллельный сдвиг кривой доходности;
- построение cashflow ladder по купонам и погашениям;
- break-even analysis по позициям;
- key rate duration (KRD) по ключевым тенорам;
- сценарный анализ решений центрального банка по ключевой ставке;
- факторное разложение P&L;
- экспорт результата в Excel с отдельными листами и графиками.

## Для чего нужен проект

Репозиторий полезен, если нужно:

- быстро собрать аналитический отчёт по портфелю облигаций;
- оценить чувствительность портфеля к движению ставок;
- посмотреть денежные потоки по периодам;
- сравнить эффект разных сценариев для кривой доходности;
- подготовить выгрузку для внутренней отчётности, презентации или инвестиционного комитета.

## Основные метрики

### Позиционный анализ

Для каждой бумаги можно считать:

- ISIN / SECID;
- рыночную стоимость;
- чистую цену и при необходимости грязную цену;
- НКД;
- доходность к погашению;
- дюрацию и модифицированную дюрацию;
- купонные характеристики;
- вклад бумаги в портфельный риск.

### Портфельные показатели

На уровне портфеля отчёт обычно содержит:

- суммарную рыночную стоимость;
- средневзвешенную доходность;
- средневзвешенную дюрацию;
- агрегированную чувствительность к ставке;
- распределение будущих платежей во времени.

### Stress testing

Поддерживается сценарный анализ изменения кривой доходности, например:

- +25 bps;
- +50 bps;
- +100 bps;
- -25 bps;
- -50 bps;
- -100 bps.

По каждому сценарию можно пересчитать стоимость бумаг и итоговый эффект на портфель.

### Cashflow ladder

Cashflow ladder показывает, когда именно портфель получает:

- купонные выплаты;
- амортизацию;
- погашение номинала;
- общий денежный поток по периодам.

Это удобно для анализа ликвидности и планирования будущих поступлений.

### Break-even analysis

Break-even analysis помогает понять, за какой срок бумага или портфель компенсирует текущую цену через купоны и возврат основного долга. Этот блок особенно полезен для сравнения бумаг с близкой доходностью, но разным профилем денежных потоков.

### Key Rate Duration

KRD показывает чувствительность портфеля не к одному параллельному сдвигу всей кривой, а к отдельным точкам по срокам, например 0.25Y, 0.5Y, 1Y, 2Y, 3Y, 5Y, 7Y, 10Y, 15Y и 20Y. Это помогает увидеть, в какой части кривой сосредоточен основной процентный риск.

### Сценарии центрального банка

Отдельный блок может моделировать решения по ключевой ставке и перенос этого решения на короткий, средний и длинный конец кривой доходности. На практике это позволяет быстро оценить, что произойдёт с портфелем при более мягком или более жёстком решении регулятора.

## Источники данных

Проект ориентирован на работу с данными MOEX ISS API:

- справочник бумаг;
- рыночные котировки;
- купоны и графики выплат;
- параметры бескупонной кривой / G-Curve;
- дополнительные сведения по инструментам.

При необходимости проект можно адаптировать и под другой источник данных, если сохранить ту же структуру входных таблиц.

## Установка

### 1. Клонировать репозиторий

```bash
git clone https://github.com/your-username/bond-portfolio-analytics.git
cd bond-portfolio-analytics
```

### 2. Создать виртуальное окружение

```bash
python -m venv .venv
source .venv/bin/activate
```

```bash

```

```bash
pip install -r requirements.txt
```

## Быстрый старт

В простом сценарии достаточно описать портфель как список позиций.

```python
POSITIONS = [
    {"isin": "SU26254RMFS1", "secid": "SU26254RMFS1", "qty": 1000, "cost_price": 85.50},
    {"isin": "SU26248RMFS3", "secid": "SU26248RMFS3", "qty": 2000, "cost_price": 87.10},
]
```

После этого запускается основной скрипт:

```bash
python main.py
```

На выходе обычно формируются:

- Excel-файл с аналитикой;
- PNG-графики для stress, cashflow ladder, KRD и сценариев ставки;
- консольный лог расчётов и предупреждений.

## Пример выходных листов Excel

Итоговый файл может включать такие листы:

- `Аналитика` — позиционные метрики;
- `Сводка` — агрегированные показатели портфеля;
- `Стресс-сценарии` — переоценка при сдвиге кривой;
- `Разложение P&L` — факторный анализ изменений стоимости;
- `CF Агрегат` — денежные потоки по периодам;
- `CF По бумагам` — детализация по выпускам;
- `Break-Even` — сроки выхода в плюс;
- `KRD` — чувствительность по тенорам;
- `Сценарии ЦБ` — анализ разных решений по ставке.

## Логика расчёта

### 1. Загрузка данных

Сначала проект получает данные по бумагам, кривой и графикам выплат.

### 2. Позиционный расчёт

Для каждой бумаги рассчитываются основные характеристики: доходность, цена, дюрация, денежные потоки.

### 3. Портфельная агрегация

Потом данные складываются на уровне всего портфеля: стоимость, дюрация, агрегированный риск, структура cashflow.

### 4. Риск-сценарии

После этого рассчитываются стресс-сдвиги и сценарии изменения ставок.

### 5. Экспорт

Финальный шаг — выгрузка в Excel и сохранение графиков.

## Настройка сценариев

Обычно параметры сценариев задаются в константах:

```python
SHIFTS = [-100, -50, -25, 25, 50, 100]

CBR_KEY_RATE = 14.25
CBR_SCENARIOS = {
    "−100 bps (агрессивное смягчение)": -100,
    "−50 bps (базовый)": -50,
    "−25 bps (осторожное)": -25,
    "0 bps (пауза)": 0,
    "+25 bps (ужесточение)": 25,
    "+50 bps (жёсткое)": 50,
}
```

Это позволяет быстро менять сценарную сетку без переписывания основной логики.

## Ограничения

Текущая версия обычно предполагает, что:

- основная зона применения — рублёвые облигации;
- расчёты чувствительны к качеству входных данных MOEX;
- отдельные блоки могут требовать ручной настройки маппинга `ISIN ↔ SECID`;
- сценарии центрального банка являются модельным приближением, а не официальным прогнозом рынка.
